## Spring Framework 

### 스프링 프레임워크의 특징 

1. 경량성
 - 기존 자바 기반의 프레임워크에 비해 크기가 작고 가볍다. 
 - EJB 객체가 아닌 POJO(Plain Old Java Object) 형태의 가벼운 객체를 관리한다.
2. 제어의 역행 (Inversion of Control, IoC)
 - 객체 생성을 자바 코드로 처리하는게 아니라, 컨테이너가 처리하게 한다. 
 - 객체와 객체 사이의 의존관계 또한 컨테이너가 처리한다. 
    #### ===> 소스에 의존 관계가 명시되지 않는다. 
 - 객체 간의 낮은 결합도를 유지할 수 있다. 
3. 관점지향 프로그래밍 (Aspect Oriented Programming, AOP)
 - 비지니스 로직과 각 비지니스 메소드의 공통 로직을 분리한다. 
 - 공통 기능을 분리하여 관리하므로 응집도가 높아진다. 
    #### ===> 유지보수의 효율을 혁신적으로 향상시킨다. 
4. 컨테이너(Container)
 - 특정 객체의 생성과 관리를 담당한다.
 - 객체 운용에 필요한 다양한 기능을 제공한다. 
 - 일반적으로 서버 안에 포함되어 배포 및 구동된다.
 - 대표적인 컨테이너로는 서블릿 컨테이너와 EJB 컨테이너가 있다. 
    #### ===> 서블릿 컨테이너를 통해 스프링 컨테이너의 동작 방식을 유추할 수 있다. 
    

### 다형성  
- 객체지향 언어의 핵심 개념 중 하나 
- 상속, 메소드 재정의(Overriding), 형변환을 이용하여 구축 
- 공통된 메소드를 가진 클래스의 상위에 인터페이스를 추가하여 각 클래스에서 인터페이스를 재정의하도록 한다. 

### 디자인 패턴 이용하기 
- 디자인 패턴을 이용하여 결합도를 낮추는 방법
- Factory 패턴을 적용하여 클라이언트에서 사용할 객체 생성을 캡슐화한다.  

###스프링 컨테이너 및 설정 파일 
- 대부분의 IoC 컨테이너는 각 컨테이너에서 관리할 객체들을 위한 별도의 설정 파일이 있다. 
- 스프링 프레임워크는 자신이 관리할 클래스들이 등록된 xml 설정 파일을 사용한다. 

#### 스프링 컨테이너의 종류 
1. BeanFactory
 - 스프링 설정 파일에 등록된 <bean> 객체를 생성하고 관리하는 가장 기본적인 컨테이너 기능을 제공한다. 
 - 컨테이너가 구동될 때가 아닌, 클라이언트가 요청할때만 <bean>객체를 생성한다. (=지연 로딩 방식) 
2. ApplicationContext
 - BeanFactory를 상속한 컨테이너 
 - BeanFactory가 <bean>객체 관리 기능 외에, 트랜잭션 관리, 메시지 기반 다국어 처리 등 다양한 기능을 지원한다.
