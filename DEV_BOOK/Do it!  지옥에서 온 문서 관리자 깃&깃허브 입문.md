- 01-1 지옥에서 온 관리자, 깃
    - 깃으로 할 수 있는 것들
        1. 버전 관리: 문서를 수정할 때마다 수정과 변경 사항을 편리하게, 구체적으로 기록  
        2. 백업하기: 자료가 유실될 위험을 대비하여 현 저장소에 있는 자료를 다른 저장소에 복제 
        3. 협업하기: 여러 사람이 함께 일할때 파일 주고받기와 수정기록 파악, 오류 발견에 용이
    - 버전 관리를 이해하지 못하면 백업을 이해할 수 없고, 백업을 이해하지 못하면 협업을 이해할 수 없기에, 깃은 꼭 1. 2. 3. 순서대로 배워야 한다.
    - 깃 허브 데스크톱, 토터스깃, 소스트리, 커맨드 라인 인터페이스 등의 프로그램을 사용

- 01-2 깃 설치하기
    - git bash 를 다운받아 (https://git-scm.com) 설치 진행

#### 현재 디렉터리 살펴보기
```bash
$ git init  --현재 폴더를 깃 저장소로 지정한다.  
$ pwd  --현재 위치의 경로
$ ls 현재  --디렉터리 안의 파일과 디렉터리 확인 
$ ls -l  --파일과 디렉터리의 상세 정보까지 표시함 
$ ls -a  --숨긴 파일과 디렉터리를 표시  
$ ls -r  --파일의 정렬 순서를 거꾸로 표시 
$ ls -t  --파일 작성 시간 순서로(내림차순) 표시 

$ clear --화면 정리
```
#### 디렉터리 이동하기
```bash 
$ cd folder --디렉터리를 이동함 
$ cd ..  --현재 디렉터리의 상위 디렉터리 
$ cd ./  --현재 사용자가 작업중인 디렉터리 
$ cd ~  --접속중인 사용자의 홈 디렉터리 
```

#### 디렉터리 생성 및 삭제
```bash 
$ mkdir --새 디렉터리를 만든다
$ rm  --디렉터리를 삭제한다 
$ rm -r  --디렉터리의 하위 디렉터리와 파일까지 삭제한다. 
```

#### Vim ex 모드 명령 
```vim
:w || :write --편집 중인 문서를 저장 
:q || :quit --편집기를 종료함 
:wq (파일) --문서를 저장하고 종료
:q! --문서를 저장하지 않고 종료
```


- git 연결 끊기
    - 수동삭제 for Windows - git 이 연결된 해당 폴더로 이동하여 .git 폴더를 삭제한다.
    - 명령어 입력 for Windows, Mac - git remote remove origin  - Remote를 해제하여 로컬과 원격 저장소의 연결을 종료시킨다.
    - 명령어를 통한 .git 파일 삭제 for Windows, Mac
        - find ./ -name ".git" | xargs rm - Rf
        - Mac의 경우 .git 파일은 숨겨져 있기 때문에 명령어로 삭제해야한다.

02-2 버전 만들기

- 파일을 수정하고 저장할 때마다 이전 내용과 수정 내용을 구분하기 위하여 파일 이름을 달리 저장한다면, 관리가 복잡하고 번거롭다.
- 파일을 변경한 후 수정 내용과  버전을 만든 시간까지 기록할 수 있는 것이 깃과 같은 버전 관리 시스템이다.
- 깃에서 버전을 관리하면 원래 파일 이름은 그대로 유지하면서  파일에서 무엇을 변경했는지를 변경 시점마다 저장할 수 있다.
- 스테이지와 커밋 이해하기
    - 작업 트리 : 파일 수정, 저장 등의 작업을 하는 디렉터리.
    - 스테이지 : 버전으로 만들 파일이 대기하는 공간
    - 저장소 : 스테이지에서 대기하고 있던 파일을 버전으로 만들어 저장하는 곳
    - 스테이지와 저장소는 눈에 보이지 않으며 숨은 파일로 존재한다.
- 작업 트리에서 빔으로 문서 수정하기
    - $ git status : 깃의 현재 상태 확인
    - $ git add 파일이름.확장자 : 수정한 파일을 스테이징하기
    - $ git commit -m "메시지" : 메시지와 함께 스테이지에 있는 파일을 커밋하기
    - $ git commit -am  "메시지" : 스테이징과 커밋 과정을 한번에 처리함

02-3 커밋 내용 살펴보기 

- $ git log : 커밋한 버전에 대한 설명 출력
- $ git diff  : 스테이지에 있는 파일과 저장소에 있는 최신 커밋을 비교하여 차이점을 보여줌 - 커밋 전 최종 검토용

02-4 버전 만드는 단계마다 파일 상태 알아보기 

- tracked / untraced 파일 : 깃이 추적하고 있는/있지 않은 파일을 나타내며 한 번도 깃에서 버전 관리를 하지 않은 파일은 untracked 로 표시됨.

    참조 :  .gitignore 파일을 만들면 특정 확장자나 제목의 파일, 또는 디렉터리를 지정하여 버전 관리에서 제외시킬 수 있다. 

- unmodified, modified, staged 상태 :  각각 수정되지 않은 상태, 수정된 상태, 스테이지에 올라간 상태를 뜻함

    참조 : $ git commit —amend 로 직전에 커밋한 메시지를 수정할 수 있다. 

02-5 작업 되돌리기 

- $ git checkout — 파일이름.확장자 : 작업 트리에서 수정한 파일을 git의 버전으로 되돌린다.
- $ git reset HEAD 파일이름.확장자 : 수정된 파일을 스테이징 했을 때, 스테이징을 취소한다.
- $ git reset HEAD^ : 깃과 스테이지를 최신 커밋을 하기 전 상태로 되돌린다.
    - —soft HEAD^ : 최신 커밋을 하기 전 상태로 작업트리를 되돌린다.
    - —mixed HEAD^ : 최근 커밋과 스테이징을 하기 전 상태로 작업 트리를 되돌린다. 옵션 없이 git reset  명령을 사용할 경우 이 옵션을 기본으로 적용한다.
    - —hard HEAD^ : 최근 커밋과 스테이징, 파일 수정을 하기 전 상태로 작업 트리를 되돌린다. 이 옵션을 사용할 경우 되돌린 내용은 복구할 수 없다.
- $git reset 커밋해시 : 특정 커밋 버전으로 되돌린다.
- $git revert  커밋해시 : 커밋 삭제하지 않고 되돌린다.
